Folder Structure

Here is the React depiction of the folder arrangement. To facilitate layout management and expedite development, we've meticulously designed partials. Incorporating a starter kit that serves as a launchpad for commencing projects from the ground up.
  • Admin
  • public
    src
    • assets            // includes all scss, fonts, images, lang, js files for the development.
      • fonts            // includes all ICONS fonts
      • images
      • scss
    • common            // Includes all the common files.
      • data            // json data used in whole admin.
    • Components            // all the layout of admin related files are here.
    • helpers
      • AuthType            // includes fakebackend.ts file.
      • jwt-token-access            // includes accessToken.ts and auth-token-header.ts file.
      • api_helper.ts
      • fakebackend_helper.ts
      • firebase_helper.ts
      • url_helper.ts
    • constants            // includes layout.ts file.
    • locales            // it contains languages json files.
    • pages            // Every single pages of Skote id described here with functionality.
    • routes            // You can find descriptions for all the route paths in allRoutes.ts file.
    • slices            // store folder used for middleware saga and reducer.
    • App.test.ts
    • App.tsx            // scss and middleware like fakebackend and firebase are imported here.
    • index.tsx            // this file is the main file which renders the app.
    • config.ts
    • i18n.ts
    • reportWebVitals.ts
    • setupTests.ts
    .env
    global.d.ts
    package.json            // all the packages are in this file and you can add new packges.
    tsconfig.json
